<objective>
Land the foundation for Phase 33's CLI surface expansion:

1. Add `exit_codes.AMBIGUOUS` for the `cg what-tests` probe-both ambiguity case (D-20).
2. Extend `cg status` output with a `repository: <uri>` line (D-15 / CLI-14).
3. Pin both behaviors with tests so subsequent plans build on a stable foundation.

**Deviation from CONTEXT.md D-20 (`AMBIGUOUS = 2`):** the value `2` is
already taken by `STALE` in `exit_codes.py` (line 7) and is actively
consumed by `cg status` (returned when the DB is stale). Assigning
AMBIGUOUS=2 would collide with STALE=2 — two semantically different
conditions sharing an integer. The next free integer after the existing
constants (SUCCESS=0..UPDATE_IN_PROGRESS=6) is **7**. This plan uses
`AMBIGUOUS = 7` and documents the deviation in the test
(`test_exit_codes_module_constants`) so any future re-read of D-20 lands
on the live source, not the prototype value.

Purpose: Unblock the 12 new q_* modules planned in 33-02/03/04 by giving
them a known-good `AMBIGUOUS` constant and an extended `cg status` to
reference in their tests. CLI-14 is delivered in this plan.

Output: AMBIGUOUS constant added; ops_status.py extended; 3 new tests.
</objective>
