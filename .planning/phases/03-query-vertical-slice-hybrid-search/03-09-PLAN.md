<objective>
Close the fourth and final SC-1 failure mode from `03-HUMAN-UAT.md`: when vault pages are TODO stubs, the librarian returns no useful excerpts, and the synthesizer (now constrained by the 03-08 no-invention rule) returns either an empty answer or a "vault does not document X" disclaimer. The lattice-wiki baseline handles this by falling back to reading source code directly. This plan ports that fallback by adding a third role (`code_reader`) and a bounded `read_file` tool, wired into `run_query` as a fan-out that fires ONLY when the regular librarian pass returns nothing useful.

Purpose: Recover answer quality on vault-thin queries (the case where the original UAT side-by-side test ran — `~/Personal/wiki/deep-agents/` has TODO-stub pages for SubagentPool topics). Per `03-HUMAN-UAT.md` recommendation, this is the medium-scope follow-on to 03-08. It is intentionally separated so the small-scope prompt fix can be measured in isolation.

Output: A `code_reader` role, a `CODE_READER_SYSTEM` prompt, a `read_file` tool with a strict allow-list, a code-fallback branch in `run_query` gated on empty librarian results, and a marker that flags the resulting answer as code-derived.
</objective>
