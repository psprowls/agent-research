Public functions to expose in commands/query.py (this plan):
- def build_index(vault_path: Path) -> None
    Builds/refreshes `.code-wiki/bm25/` and `.code-wiki/search.db` under vault_path.
    Uses sha256 incremental rebuild (D-02). Calls BedrockEmbeddings.embed_query per
    new/changed page sequentially (parallel embedding is a future optimization).

- def bm25_query(query_text: str, vault_path: Path, top_k: int)
      -> tuple[list[str], list[float]]
    Loads BM25 + tokenizer state from .code-wiki/bm25/, tokenizes with
    update_vocab=False, returns (page_paths, bm25_scores) at top_k.

- def _cosine_search_sqlite(vault_path: Path, query_vec: list[float], top_k: int)
      -> list[tuple[str, float]]
    Linear cosine scan of .code-wiki/search.db (path, embedding).
    Opens + closes connection per call (no module-level state).

- def _rrf_fuse(bm25_ranks: dict[str, int], embed_ranks: dict[str, int],
                k: int = 60) -> dict[str, float]
    Reciprocal Rank Fusion: 1/(k + rank_bm25) + 1/(k + rank_embed).
    Missing pages use rank = n + k as the sentinel "unranked" position.

- def _build_tokenizer() -> Tokenizer
    Returns a bm25s.tokenization.Tokenizer instance with:
      lower=True
      splitter=r"[a-zA-Z0-9][a-zA-Z0-9_\-']+"
      stopwords=list(_STOPWORDS)
    Used at both build and query time (vocab is loaded at query time).

- def _discover_pages(vault_path: Path) -> list[tuple[str, str]]
    Walks vault_path.rglob("*.md") and returns [(rel_path_str, text), ...]
    Skips: rel.name in {"index.md", "log.md"} or any part starts with "."
    Paths are POSIX-normalized (replace("\\", "/")).

SQLite schema (DDL — copy verbatim into query.py):
  CREATE TABLE IF NOT EXISTS pages (
      path         TEXT PRIMARY KEY,
      content_hash TEXT NOT NULL,
      embedding    BLOB NOT NULL
  )
  PRAGMA journal_mode=WAL;

Upsert pattern:
  INSERT OR REPLACE INTO pages (path, content_hash, embedding) VALUES (?, ?, ?)

Embedding BLOB encoding:
  blob = struct.pack(f"{len(vec)}f", *vec)        # write
  vec  = struct.unpack(f"{len(blob)//4}f", blob)  # read
  Expected dims: 1024 (Titan v2 default, per RESEARCH §"Standard Stack" + AI-SPEC §4)

bm25s usage (Context7 /xhluca/bm25s, verified in RESEARCH Q1+Pitfall 7):
  # Build:
  tokenizer = Tokenizer(lower=True, splitter=_TOKEN_RE_PATTERN, stopwords=list(_STOPWORDS))
  corpus_paths = [...]   # list of POSIX page paths
  corpus_texts = [...]   # list of page text bodies
  corpus_tokens = tokenizer.tokenize(corpus_texts)
  retriever = bm25s.BM25(method="lucene", k1=1.5, b=0.75)
  retriever.index(corpus_tokens)
  retriever.save(str(bm25_dir), corpus=corpus_paths)  # corpus = paths so retrieve returns paths
  tokenizer.save_vocab(str(bm25_dir))
  tokenizer.save_stopwords(str(bm25_dir))

  # Query:
  retriever = bm25s.BM25.load(str(bm25_dir), load_corpus=True)
  tokenizer = _build_tokenizer()
  tokenizer.load_vocab(str(bm25_dir))
  tokenizer.load_stopwords(str(bm25_dir))
  query_tokens = tokenizer.tokenize([query_text], update_vocab=False)
  results, scores = retriever.retrieve(query_tokens, k=top_k)
  # results.shape == (1, top_k); results[0, i] is the corpus item (page path string)

BedrockEmbeddings construction (langchain-aws, verified in RESEARCH §"Pattern 3"):

      region_name="us-east-1",
      normalize=True,                              # L2 normalize
  )
  vec = embeddings.embed_query(text)               # returns list[float], 1024 dims

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Vault filesystem | Markdown pages read from arbitrary disk locations under `vault_path` |
| Bedrock API | `BedrockEmbeddings` sends page content to AWS for embedding |
| SQLite DB | Local file; multiple readers possible during query while index rebuilds |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | Path traversal via vault_path causing reads outside intended dir | mitigate | `_discover_pages` uses `vault_path.rglob("*.md")` which is rooted at vault_path; `_workspace.resolve_wiki_and_repo()` (called in Plan 03) calls `Path.resolve()` first. Skip-list prevents reading `.git/`, `.code-wiki/`, etc. |
| T-03-05 | Information Disclosure | Vault page content sent to AWS Bedrock for embedding | accept | Documented behavior: Bedrock-only project; vault is owned by user (Pat); no PII per Section 1b regulatory analysis |
| T-03-06 | Denial of Service | Oversized vault page causing embed_query token-limit error | mitigate | Truncation guard in Plan 03 `drill_page`; for embeddings, Titan v2 silently truncates at 8192 tokens (no error); log at WARNING in build_index if any page text exceeds 32000 chars |
| T-03-07 | Tampering | Concurrent SQLite reader during rebuild | mitigate | WAL mode (`PRAGMA journal_mode=WAL`) enables safe concurrent reads while writes happen; connection opened+closed per call (no leaked transactions) |
</threat_model>
