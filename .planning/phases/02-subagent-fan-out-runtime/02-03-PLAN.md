<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `trace <file>` CLI input → filesystem | Caller-controlled file path; Typer's Path type validates existence semantics; missing-file path returns exit code 1 with stderr message, not a Python traceback |
| JSONL line → `json.loads` | The JSONL file is operator-controlled (output of our own pool); malformed JSON would raise `json.JSONDecodeError` and propagate as a Python error — acceptable because the file is not externally sourced |
| Integration tests → real Bedrock | AWS creds come from env or IAM role; tests only invoke `bedrock:InvokeModel`; failures surface to the test report, never persisted |
| Trace viewer stdout → user terminal | All output via `typer.echo`; no shell interpolation of record fields |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Tampering | Malformed JSONL line crashes trace viewer mid-file | accept | JSONDecodeError on a bad line aborts rendering of remaining records; rare edge case; user can re-run after inspecting the failing file. Acceptable for a single-operator tool. |
| T-02-03-02 | Information Disclosure | Trace viewer prints `item_id` strings that may contain secrets | accept | item_id is set by the caller; if a caller puts secrets in items, they appear in trace files regardless. Documented in pool docstring (Plan 02 threat T-02-02-02). |
| T-02-03-03 | Denial of Service | Very large trace file exhausts memory via `file.read_text().splitlines()` | accept | Practical trace files are <10 MB; if this becomes an issue Phase 4 can switch to line-by-line iteration. Not a Phase 2 concern. |
| T-02-03-04 | Information Disclosure | Integration test prompts leak via Bedrock logs / CloudTrail | accept | Test prompts are deliberately trivial ("Say: ok-{item}", "Reply with exactly: ok-{item}", "Step N: respond with 'ok'"); no PII, no secrets. CloudTrail logs are Pat's account. |
| T-02-03-05 | Elevation of Privilege | Integration test running without CODE_WIKI_RUN_INTEGRATION gates triggers unexpected AWS costs in CI | mitigate | All three tests carry `@INTEGRATION_GATE` (`pytest.mark.skipif(not os.environ.get(...))`); without the env var they SKIP, never invoke Bedrock. Verified by the no-env-var verify command exiting 0. |
| T-02-03-06 | Tampering | Typer `Path` argument with shell metacharacters | accept | Typer / Click handle argv parsing; we never shell out. No interpolation risk. |

**Residual risk:** low
</threat_model>
