Constants:
- `REPO_ROOT` — resolve from `Path(__file__).resolve().parents[<n>]` (existing pattern; verify by reading the current file).

- `PROMPTS_DIR = REPO_ROOT / "agents/graph-wiki-agent/src/graph_wiki_agent/prompts"` (parent of FRAGMENT_DIR — scan only `prompts/*.py` direct children, not recursive)
- `ALLOWED_PREFIXES` — tuple of three values:
  - `"plugins/graph-wiki/"`
  - `"packages/workspace-io/src/workspace_io/assets/CLAUDE.md.template"` (exact-string allowed; treated as a literal whole-path)
  - `"agents/graph-wiki-agent/src/graph_wiki_agent/prompts/sources/"`
- `_PROVENANCE_RE = re.compile(r"^# Source: (?P<source>\\S+)(?: §(?P<sections>.+))?\\s*$", re.MULTILINE)`

Helper: `slugify(heading: str) -> str` per PATTERNS § Pattern 6 — lowercase the heading text after stripping leading `#`+space; strip punctuation except hyphens/underscores; replace whitespace runs with single hyphen; no trailing hyphen. Test in module against the D-03 example: `"### 4. Write the source summary"` → `"4-write-the-source-summary"`, AND `"### Pass 2 — Semantic (read and think)"` → `"pass-2--semantic-read-and-think"` (em-dash gets stripped; double-hyphen preserved). Use unicodedata or explicit `str.translate` — em-dash is U+2014.

Helper: `extract_headings(path: Path) -> set[str]` — read the file, find every `^#+ ` line, return the set of slugs.

Helper: `extract_section_tokens(path: Path, section_slug: str) -> set[str]` — read the file, locate the heading whose slug == section_slug, extract the body until the next `^#+ ` heading. From the body extract:
  - `\b[A-Z][A-Za-z0-9_]+(?:\s+[A-Z][A-Za-z0-9_]+)*\b` (capitalized noun phrases ≥3 chars)
  - `\b[a-z_]+_[a-z_]+\b` (snake_case identifiers)
  - `` `[^`]+` `` (backticked code spans)
Apply English stoplist (`The`, `When`, `If`, `A`, `An`, `Or`, `And`, `Not`, `In`, `On`, `Of`, `To`, `For`, `By`, `With`, `As`, `Is`, `Are`, `Be`, `This`, `That`, `These`, `Those`) to drop pure-stopword capitalized terms.

Helper: `find_constant_after_comment(file_text: str, comment_line_idx: int) -> str` — locate the Python string constant (triple-quoted or single-quoted assignment) that begins within ~5 lines after the `# Source:` comment. Return the string content.

Tests:
1. `test_every_source_path_uses_allowed_prefix` — scan FRAGMENT_DIR/*.py and PROMPTS_DIR/*.py (direct children only, exclude `_fragments/` since it's already covered, exclude `sources/` since it's .md not .py), assert every captured `source` starts with one of `ALLOWED_PREFIXES` or equals the workspace-io template literal.
2. `test_every_source_section_resolves_to_a_heading` — for each `# Source: <path> §<section>` capture, parse the section list (comma-separated `§`-prefixed names), and for each section assert `slugify(section_name) in extract_headings(REPO_ROOT / path)`.
3. `test_semantic_overlap_meets_threshold` — for each `# Source: <path> §<sections>` capture and each section, compute `len(section_tokens ∩ constant_tokens) / len(section_tokens) >= 0.70`. The constant_tokens set is extracted from the Python string constant immediately following the comment using the same extractor. If `len(section_tokens) == 0` (degenerate section), skip with a pytest.skip + warning so the gate isn't silently strengthened by empty pools.

D-09 carry: if a fragment fails the 70% gate, the response is to widen the fragment's Python string constant content (the canonical citation), NOT relax the threshold. This plan does not allow threshold reduction.

Path-resolution note: `_PROVENANCE_RE` captures the raw source path. Map to filesystem in one of two ways:
- if it starts with `plugins/graph-wiki/` or `agents/graph-wiki-agent/src/graph_wiki_agent/prompts/sources/` → `REPO_ROOT / source`
- if it equals the literal `packages/workspace-io/src/workspace_io/assets/CLAUDE.md.template` → same `REPO_ROOT / source`
