## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem → classify() | pyproject.toml / package.json content is read by upstream manifest readers; classify() receives only the sanitized info dict |
| classify() return → emit loop | Plan 02 wires the return tuple into the SQLite emit path; the _VALID_APP_KINDS gate in this plan rejects malformed app_kind before it reaches the DB |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-50-01-T | Tampering | classification.py classify() | mitigate | Pure function with no I/O; cannot be tampered with via external state; _VALID_APP_KINDS frozenset gate catches programmer typos at write time |
| T-50-01-I | Information Disclosure | _read_pyproject / _read_package_json extensions | accept | These readers already run on locally-checked-out manifests; adding presence booleans does not enlarge the read surface |
| T-50-01-D | Denial of Service | (pkg_dir / "index.html").exists() filesystem check | accept | Single stat() call per JS manifest; bounded by manifest count which is already bounded in _discover_manifests |
| T-50-01-V | Input Validation | _VALID_APP_KINDS gate inside classify() | mitigate | Assert-or-raise on return; covered by the precedence multi-signal test |
</threat_model>
