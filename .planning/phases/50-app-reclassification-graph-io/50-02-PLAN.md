<objective>
Wire `classify()` into the `packages.refresh` emit loop and implement the D-06 in-place kind-flip mutation so a `Package` node becomes an `App` node (and vice versa) without changing its row id. Update the contains-edge emission to use `new_kind` instead of the hard-coded `"package"` string, and add integration tests that prove the round-trip pkg→app→pkg flip preserves inbound edge FKs. Implements decisions D-04, D-05, D-06, D-07 from `50-CONTEXT.md`; addresses APP-01, APP-02, APP-06.

Purpose: This is the load-bearing change of Phase 50. Plan 01 produced the pure primitives; this plan moves them into the live scanner so `cg update` actually classifies apps. The D-06 in-place UPDATE is the trickiest part of the phase — it must run inside the same transaction that wraps `packages.refresh` so SQLite's read-your-own-writes semantics let the subsequent `upsert._upsert_node` call find the just-updated row by its NEW (kind, name, path) key.

Output: One modified source file (`packages.py`) and additional integration tests in `tests/test_packages.py`. No new files.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem manifest → refresh emit loop | pyproject.toml and package.json content arrives via _read_pyproject / _read_package_json; classify() and the D-06 UPDATE both consume this trusted-but-attacker-influenced data |
| Pre-existing nodes row (other-kind probe) → in-place UPDATE | The probed row id originates from upsert._node_id (parameterized query, no SQL injection); the UPDATE uses parameterized SQL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-50-02-T | Tampering | D-06 UPDATE statement | mitigate | All four UPDATE parameters bound via sqlite3 ? placeholders; new_uri composed from app_uri()/pkg_uri() which f-string from RepoContext strings (already a trust boundary upstream); attrs_for_db serialized via json.dumps with sort_keys=True for stable output |
| T-50-02-R | Repudiation | Cross-run kind flips | accept | No audit log requirement; flip is idempotent and reversible; the kind+uri columns ARE the audit record |
| T-50-02-I | Information Disclosure | classify() reads (pkg_dir / "index.html").exists() | accept | Same trust boundary as the manifest reader; bounded by manifest discovery scope |
| T-50-02-D | Denial of Service | _node_id probe per manifest iteration | accept | Single indexed lookup per manifest; manifest count is bounded by _discover_manifests scan; Claude's-Discretion default in CONTEXT.md selected per-manifest SELECT for code locality |
| T-50-02-E | Elevation of Privilege | The in-place UPDATE bypasses upsert.upsert_records | mitigate | UPDATE only mutates kind/uri/attrs_json on a row already owned by the manifest discovery path (probed by name+path+kind); cannot touch unrelated rows |
</threat_model>
