<objective>
Stop `graph-io`'s scanner from double-classifying a workspace package. In
`packages.refresh()`, a dependency name that matches a workspace `package`/`app` must NOT
produce a `dependency` node; instead the internal package→package relationship is emitted as a
new `depends_on_package` edge (src=consumer, dst=internal package), and the existing `used_by`
edge for that pair is retargeted from the (now-suppressed) `dependency` node to the real
package/app node. External (non-workspace) dependencies are unaffected.

Purpose: today `refresh()` accumulates EVERY declared Python dependency into `dep_acc` —
including names that are themselves workspace members (e.g. `graph-io`) — so a workspace
package gets both a `package` node and a `dependency` node, and downstream a stray
`dep_graph-io.md` entity page. This is the CLASS-01 bug.

Output: a `refresh()` that suppresses the `dependency` node for any normalized workspace name
(D-01/D-02/D-03), emits a `depends_on_package` edge for the internal relationship (D-04/D-06),
keeps a retargeted `used_by` edge resolved to the target's stored kind (D-07), and unit tests
covering suppression, both edges, dedupe preservation, and the external-dep regression.
</objective>

<threat_model>
## Trust Boundaries

This plan touches only `graph-io`'s manifest-scanning code path (`refresh()`), which reads
local workspace manifest files (`pyproject.toml`/`package.json`) already present in the repo and
writes nodes/edges into the local sqlite graph inside the caller's transaction. No network, no
untrusted input, no new execution surface, no new external data flow.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-55-01-T1 | Tampering | Over-broad suppression silently drops legitimate external `dependency` nodes (e.g. a typo in the name set, or normalization collapsing distinct names) | mitigate | Suppression is gated on membership in the workspace-name set only; the Task 2 regression test asserts external deps (boto3) keep their `dependency` node + `used_by` edge. Normalization (lower + `-`→`_`) mirrors the existing `import_scan` convention and is unit-covered by the hyphen/underscore mismatch case. |
| T-55-01-T2 | Tampering | Retargeted `used_by` / new `depends_on_package` dst dangles when the internal target is an `app` rather than a `package` | mitigate | Dst kind is resolved from the workspace `name -> (stored_kind, rel_path)` map (D-07, mirroring derived_edges.py:148-153); Task 2 optionally exercises an `app` target and asserts the joined dst node kind is `package`/`app`, never a missing/`dependency` row. |
| T-55-01-T3 | Denial of Service | Edge dedupe regression doubles edges per relationship, inflating the graph | mitigate | The existing `seen_edges` dedupe is preserved and applied to both new edges; Task 2 includes an explicit dedupe assertion mirroring the existing dedupe test. |
