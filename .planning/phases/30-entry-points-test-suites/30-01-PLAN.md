<objective>
Amend the `_is_test_path` heuristic in
`packages/graph-io/src/graph_io/structural_nodes.py` to add the **src-override**
rule from CONTEXT.md D-01: a file inside any Python Package's import root, or
inside a JS/TS Package directory but outside any `tests/`/`__tests__/` subdir,
is `is_test=false` regardless of filename — even when the filename matches
`test_*.py` / `*_test.py` / `*.test.{ts,js}` / `*.spec.{ts,js}`.

The `tests/` directory branch remains authoritative — files under a `tests/`
ancestor stay `is_test=true`. Only the *filename-only* branch of the original
D-09 heuristic now gates on whether the file is inside a Package import root.

`structural_nodes.emit`) to a module-level helper so Plan 30-02
(`entry_points.emit`) and Plan 30-03 (`test_suites.emit`) can reuse the
deepest-Package-wins lookup without duplicating it.

This plan is a Phase 30 hotfix to a Phase 29 deliverable. Phase 29 plan
29-03 shipped the original D-09 heuristic in commit `fc0474d` before Phase 30
context was captured; D-02 specifies this amendment lands in Phase 30 since
Phase 29 had already shipped.

Output: A purely additive amendment — no Phase 29 test breaks, three new
unit tests pass, and the hoisted `_owning_package` is callable from outside
`structural_nodes.emit`.
