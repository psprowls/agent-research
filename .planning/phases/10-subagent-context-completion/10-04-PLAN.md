<objective>
Add the project-context renderer `prompts/project_context.py::render_project_context(wiki_path) -> str`. It reads `wiki/CLAUDE.md` (falling back to `wiki/AGENTS.md`), parses the embedded layout block via the existing `vault_io.layout_io.read_layout`, and emits a compact deterministic rendered block (~30 lines) covering layout containers, style rules, and log format. Returns `""` when neither schema file exists.

Purpose: This is the runtime half of the spike-001 solution. Static fragments cover SKILL-level facts; this renderer delivers project-specific layout pins, style guidance, and log format to subagents at command entry. Output: One new module under `prompts/` exporting `render_project_context`, plus a new test module under `tests/prompts/` covering the four cases (present-CLAUDE, present-AGENTS, missing-both, snapshot stability).

    Public function: `render_project_context(wiki_path: Path) -> str`
    - For each candidate name in the tuple `("CLAUDE.md", "AGENTS.md")` (in that exact order — CLAUDE.md takes priority), check whether `wiki_path / name` exists.
    - On the first existing candidate, call `read_layout(schema)` to parse the layout block, then call a private `_render(layout, schema)` helper that returns the rendered string.
    - If neither candidate file exists, return `""`.
    - No `try/except` swallowing — if `read_layout` raises (it doesn't currently), let it propagate.

    Private helper: `_render(layout: dict | None, schema_path: Path) -> str`
    - Composes the rendered block as a single string with three sections in this order:
      1. `## Project layout (parsed from wiki/<filename>)` — list each container as `- <vault_dir> → <classification> (<children_count> children)` if `children_count` is present, else `- <vault_dir> → <classification>`. If `layout` is None or has no `containers`, emit `- (no layout block detected)`. Sort containers by `vault_dir` alphabetically so ordering is deterministic regardless of YAML key order.
      2. `## Project style (wiki/<filename> §Style)` — extract the `## Style` section body from the raw markdown via a simple heading-walk (read `schema_path.read_text()`, find the `## Style` heading line, collect subsequent non-heading lines until the next `## ` heading). If no `## Style` section exists, omit the section entirely.
      3. `## Log format (wiki/<filename> §Log format)` — same heading-walk approach for the `## Log format` section. Omit if absent.
    - Concatenate sections with `"\n\n"`. Strip trailing whitespace once at the end.
    - Filename in the section headers should match what was actually read (`CLAUDE.md` or `AGENTS.md`). Use `schema_path.name` to fill the placeholder.

    Determinism requirements (per CONTEXT.md §Project-context renderer LOCKED): sort containers by `vault_dir`. No timestamps, no random ordering, no Path.iterdir() (which can be filesystem-order-dependent).
