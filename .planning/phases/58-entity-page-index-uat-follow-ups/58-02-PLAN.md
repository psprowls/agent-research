<objective>
Fix the scan-side half of Item #3 (SC#3b, D-07/D-08/D-09): rename package-owned `test_suite` nodes from `Path(rel_path).name` (always `'tests'`, causing the index fan-out) to a unique package-qualified name `<owner_name>-<suite_kind>-tests`. There are FOUR mutation points in `test_suites.py` (RESEARCH §"Summary of scan-side changes"), all of which must change consistently, and the `kind_attr`/`suite_name` computation order must swap (the new name depends on `kind_attr`). Repository-owned suites already use the full `rel_path` (unique) and are NOT renamed (D-09).

This plan also rebaselines the graph-io fixtures (D-10) and resolves the stale-node landmine (RESEARCH OQ#3): the fix is a fresh `cg update --full`, because `update.py` already issues a full-scan DELETE of non-package/app/builtin path-bearing nodes (line 285) before re-emitting — incremental updates leave stale `tests`-named nodes, full updates do not.

Purpose: Give every test suite a unique, human-meaningful name so the index renderer (Plan 03) can resolve consumer packages per-suite without fan-out, and so `cg describe-suite <name>` names a single suite.

Output: Edited `test_suites.py` (four locations + reorder), updated `test_test_suites.py` assertions, a verified SC#3b uniqueness check.
</objective>
