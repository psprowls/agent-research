D-09 layer 3 — **the main event**. Sweep imports + symbol renames + string literals + trace-dir path fragments inside the agent package (`agents/graph-wiki-agent/src/` + `agents/graph-wiki-agent/tests/`). This is the layer where pytest-green resumes; D-11 gate is `uv sync && uv run pytest agents/graph-wiki-agent/tests/ -m "not integration"`.

Three substitution classes per 21-PATTERNS.md `python-module-sweep`:

1. **Snake imports** — `from code_wiki_agent.X` → `from graph_wiki_agent.X`; `from code_wiki_mcp.X` → `from graph_wiki_mcp.X`; bare `import code_wiki_agent` → `import graph_wiki_agent`.
2. **Kebab user-facing strings** — `"code-wiki-agent ..."` in print/log/error/CLI-help → `"graph-wiki-agent ..."` (D-03 touches tracebacks too — consistency over churn-avoidance).
3. **Internal identifiers** — any class/func/var matching `CodeWiki[A-Z][A-Za-z]*` → `GraphWiki*` (e.g., `CodeWikiAgentError` → `GraphWikiAgentError`).
4. **Trace-dir path fragment** — `.code-wiki/` → `.graph-wiki/` (folded in; the `trace-dir-reference` class for files inside the agent package per D-04).

Per 21-PATTERNS.md: "Phase 21 has a cleaner pattern [than Phase 12 lattice]. Planner may script the sweep" — the four search tokens (`code_wiki_agent`, `code_wiki_mcp`, `code-wiki-agent`, `code-wiki-mcp`) do not collide with other identifiers, so scripted `sed` is acceptable here.

Recommended commit split inside this plan (two atomic sub-commits for revertability):
- **Task 1 commit:** snake imports + identifier renames + trace-dir path fragment sweep in `src/`. Gate: pytest green.
- **Task 2 commit:** kebab user-facing string literals + tests directory sweep + trace-dir fragment in tests. Gate: pytest green.

Both sub-commits live on `rename-21`. SP-3 ("no fix-forward") applies between them: if Task 2 breaks pytest, `git reset --hard HEAD~1`, diagnose, retry.
