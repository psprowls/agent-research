## Phase Boundary

Mechanical rename of the agent package from `code-wiki-agent` to `graph-wiki-agent` across the full repository — directory names, Python module names, console scripts, internal symbols, user-facing strings, trace dir, plugin shell-out invocations, tests, planning docs, and active documentation.

**In scope:**
- `agents/code-wiki-agent/` → `agents/graph-wiki-agent/` (`git mv`)
- `src/code_wiki_agent/` → `src/graph_wiki_agent/`; `src/code_wiki_mcp/` → `src/graph_wiki_mcp/`
- `pyproject.toml` `name = "code-wiki-agent"` → `"graph-wiki-agent"`
- Console scripts: `code-wiki-agent` → `graph-wiki-agent`, `code-wiki-mcp` → `graph-wiki-mcp`
- All import paths (`from code_wiki_agent.X` / `from code_wiki_mcp.X`) in src + tests
- All user-facing string literals (CLI help, log messages, error messages, prints) referencing the old name
- Trace directory `.code-wiki/traces/` → `.graph-wiki/traces/`
- Plugin shell-out scripts (`plugins/graph-wiki/skills/graph-wiki/scripts/*.py`) that invoke `["code-wiki-agent", ...]`
- Repo `README.md`, `CLAUDE.md`, plugin `README.md`/`CLAUDE.md`, `plugins/graph-wiki/.claude-plugin/plugin.json`
- All 188 `.planning/` files (per D-04 override): STATE.md, ROADMAP.md, REQUIREMENTS.md, PROJECT.md, prior phase CONTEXT/PLAN/VERIFICATION docs, RETROSPECTIVE.md, intel/stack.json, milestone archives
- `.claude/skills/spike-findings-deep-agents/SKILL.md` + active reference files
- Extend `scripts/check-brand.sh` + `.brand-grep-allow` to enforce the rename going forward

**Out of scope (explicit):**
- `graph-wiki/wiki/` content (companion pages, agent dir naming) — wiki is regenerated by `/graph-wiki:scan`; touching it here invites divergence
- Raw spike sources under `.planning/spikes/{001,002}/sources/**` — historical material
- Backwards-compat shims (no console-script alias, no deprecation stub)
- `models.toml` / config file renames — unrelated to package identity
- Wiki vault directory rename (`graph-wiki/wiki/agents/code-wiki-agent/`) — addressed by future scan, not this phase

<decisions>
## Implementation Decisions

### Rename Surface
- **D-01:** Full rename — directory + Python packages + console scripts. Mirrors plugin port pattern; single source of truth for the brand.
- **D-02:** Test directory moves with the package (`agents/graph-wiki-agent/tests/`); test imports updated. Also rename any test files that literally embed the old slug in the filename (spot-check during planning confirmed `test_command_overrides.py`, etc. are behavior-named, not pkg-named — likely no filename renames needed, but planner must verify with `find agents/code-wiki-agent/tests -name '*code_wiki*'` before locking).
- **D-03:** Full sweep including string literals in `print` / log messages, CLI help text, error messages. Internal class/function names that start with `CodeWiki` get renamed (e.g., `CodeWikiAgentError` → `GraphWikiAgentError`). Touch tracebacks; consistency over churn-avoidance.
- **D-04:** Rename trace directory `.code-wiki/traces/` → `.graph-wiki/traces/`. Existing local trace history orphans (ephemeral debug output, not committed).

### Scope Boundary
- **D-05:** Sweep ALL 188 `.planning/` files including historical phase docs, archived milestones, RETROSPECTIVE.md. User override of the default "active docs only" recommendation. Rationale: full historical consistency — a future grep of `.planning/` for project-name references should return one name. Acknowledged trade-off: alters historical record vs. git log/commit messages (which retain the old name immutably).
- **D-06:** Skip `graph-wiki/wiki/` content entirely. Wiki is healed via next `/graph-wiki:scan` run, not via direct edits.
- **D-07:** Update `.claude/skills/spike-findings-deep-agents/SKILL.md` + active references; leave `.planning/spikes/{001,002}/sources/**/*.md` as historical raw spike material.

### Cutover Strategy
- **D-08:** Work in a git worktree. Main checkout stays usable during the multi-commit rename. Use `git worktree add ../deep-agents-rename rename-21` (or planner's preferred naming).
- **D-09:** Staged commits per layer, each landing green:
  1. `git mv` directories (`agents/code-wiki-agent/` and `src/` subdirs) — preserves git history for moved files
  2. `pyproject.toml` `name` + console scripts + `uv.lock` regenerate
  3. Sweep imports + symbol renames + string literals inside the agent package
  4. Sweep plugin shell-out scripts + any other consumer (tests, integration gates)
  5. Sweep `.planning/` + repo docs + `scripts/check-brand.sh` extension + `.brand-grep-allow` entries
- **D-10:** Hard cut, no shim. Aligns with CLAUDE.md "Avoid backwards-compatibility hacks" + Phase 20's precedent (hard-deleted `WikiConfig.models_path` / `--config` / `CODE_WIKI_CONFIG`). User reinstalls via `uv sync`.
- **D-11:** Per-commit gate: `uv sync && uv run pytest agents/graph-wiki-agent/tests/ -m "not integration"` must pass (or equivalent scope) before each commit lands. Integration / live-Bedrock tests gated behind their existing env flags; not required per intermediate commit.

### Enforcement Gate
- **D-12:** Extend existing `scripts/check-brand.sh` to grep for `code-wiki-agent` / `code_wiki_agent` / `code-wiki-mcp` / `code_wiki_mcp` in addition to the lattice pattern. Single gate, single `.brand-grep-allow` allowlist file.
- **D-13:** Allowlist stays tight — only references that would actively break things if rewritten (e.g., commit SHA references in PROJECT.md decision logs that quote a commit message with the old name). All 188 `.planning/` files getting swept means most historical references are eliminated rather than allowlisted.

### Claude's Discretion
- Concrete commit messages and SHA-stable commit ordering inside D-09's 5 layers.
- Whether to introduce a temporary `agents/graph-wiki-agent/tests/conftest.py` adjustment if test discovery breaks mid-rename — planner judges based on actual test layout.
- Exact regex to add to `check-brand.sh` — recommended: anchored on word boundaries to avoid false positives (e.g., URLs, future docs that quote the rename history).

### Folded Todos
None — the three Phase 21 todo matches (`fix-bedrock-count-tokens-api-shape`, `rename-graph-wiki-init-command`, `fix-workspace-repo-resolution`) are already assigned to Phases 17/18 per STATE.md; they keyword-matched but are unrelated.

## Existing Code Insights

### Reusable Assets
- **`scripts/check-brand.sh` + `.brand-grep-allow`** (Phase 12 artifacts) — extend with additional pattern; reuses the same exit-non-zero-on-unallowlisted-hit mechanic that already runs as a pytest gate.
- **Phase 12 lattice → graph-wiki sweep PLAN.md** — direct template for staged-commit cadence (precedent for grepping packages/agents/plugins/.planning + atomic commits).
- **`uv` workspace** — `pyproject.toml` `[tool.uv.sources]` declares dependents as `{ workspace = true }`; only the package's own `name` field matters for the rename. Other workspace members (`vault-io`, `model-adapter`, etc.) do NOT need their pyproject.toml touched.

### Established Patterns
- **Hard-cut deletion precedent (Phase 20)** — `WikiConfig.models_path`, `--config`, `CODE_WIKI_CONFIG` deleted with no shim; same posture applies here for the console-script rename.
- **`git mv` for directories then sweep imports** — preserves blame/history for moved files (vs. `mv` + new file creation).
- **Plugin shell-out shape (`SO-01`, Phase 14)** — plugin scripts use `uv run --project "$DEEP_AGENTS_ROOT" python3 -m ...` for module-level invocations; the literal `["code-wiki-agent", ...]` form is direct console-script invocation. After rename, the console scripts will need to be available on `$PATH` (via `uv sync`); planner should confirm any docs or local-dev setup notes reflect the new names.

### Integration Points
- **Plugin → agent CLI boundary** — `plugins/graph-wiki/skills/graph-wiki/scripts/*.py` shell out to the renamed `graph-wiki-agent` console script. This is the only inter-package consumer that hard-codes the binary name.
- **MCP server entry point** — `code-wiki-mcp` is registered with the host (DeepAgents CLI, Claude Code). After rename, host configs may need to point at `graph-wiki-mcp`. Planner should confirm whether any committed host config files in the repo reference the old name.
- **Trace renderer (`code-wiki-agent trace`)** — moves to `graph-wiki-agent trace`; trace files in `.code-wiki/traces/` move to `.graph-wiki/traces/`; planner should grep for any other `.code-wiki/` path references.
