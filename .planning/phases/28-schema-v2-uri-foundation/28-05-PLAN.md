| filesystem → unlink | `db_path` is computed from `graph_dir(workspace)` which is workspace-resolved upstream; no user-supplied path crosses into the unlink. |
| concurrent updater → unlink race | Two `cg update --full` processes racing on the same workspace could both pass the v1-probe, then one unlinks while the other reopens. |
