<objective>
Extend `_upsert_node` (and its `_insert_node` helper) in `packages/graph-io/src/graph_io/upsert.py` so that when an incoming `GraphNode`'s `attrs` dict contains a `"uri"` key, the value is popped out BEFORE `_serialize(attrs)` is called and written to the dedicated `uri` column on the `nodes` table. This is the PITFALL 4 lock — the test that proves URIs do not contaminate `attrs_json` ships in this plan, before any emitter writes a URI.

Purpose: PITFALLS.md pitfall 4 is the single most expensive mistake the planner can let through: if `uri` lands in `attrs_json`, every downstream query has to JSON-extract it, every emitter has to remember to re-pop it, and the schema column becomes a lie. Locking it here, with a sentinel test, eliminates that failure mode permanently. Depends on Plan 01 (column must exist before this code can write to it).

Output: Modified `upsert.py` (~6-line change to `_upsert_node` + INSERT/UPDATE SQL strings), one new sentinel test in `test_upsert.py`.
</objective>
