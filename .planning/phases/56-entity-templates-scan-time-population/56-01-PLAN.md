<objective>
Make the scanner substitute `{{...}}` data tokens in entity-page bodies at render time (SCAN-01)
and write a fill-when-empty `summary:` frontmatter field derived from the graph node's
description (SCAN-02). Reuse the existing `init_vault.render_template()` `.replace()` mechanism
(D-02) — no Jinja. Honor the two-token rule (D-01): substitute only `{{...}}`, never `<...>`.
Unfilled tokens and empty descriptions resolve to visible `TODO:` markers (D-03/D-12), so no raw
`{{...}}` and no empty `summary:` survive.

Purpose: today `_render_entity_page` emits the template body verbatim (no substitution), so
generated pages show literal `# {{package_name}}` after Plan 02's token conversion; and no page
carries a `summary:`. This plan wires substitution and the summary derivation/merge so generated
pages have real headings and a populated summary that survives human edits on re-scan.

Output: `_render_entity_page` substitutes `{{...}}` from a node-derived variable map;
`scanner_frontmatter_for_node` emits `summary` from `node.attrs["description"]` uniformly;
`merge_frontmatter` special-cases `summary` as fill-when-empty; unit + integration tests cover
substitution, the TODO fallback, and both summary directions (preserve human / fill empty).
</objective>

<threat_model>
## Trust Boundaries

This plan edits `wiki-io`'s scan-time rendering/merge code, which reads local graph node data and
local template assets and writes markdown into the user's own vault. No network, no untrusted
input. The substitution is a literal string `.replace`, not an eval/format — no template-injection
surface (a malicious `{{...}}` token would simply not match the variable map and become a TODO
marker).

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-01-T1 | Tampering | A `{{var}}` token survives unsubstituted, failing SCAN-01 | mitigate | D-03 residual-token-to-TODO conversion guarantees no raw `{{...}}` survives; Task 3's unit + integration tests assert zero `{{` in rendered output. |
| T-56-01-T2 | Tampering | summary added to SCANNER_OWNED_KEYS clobbers a human-edited summary on re-scan | mitigate | D-07 fill-when-empty special-case in merge_frontmatter (NOT SCANNER_OWNED_KEYS); Task 3 asserts the preserve-human direction; a comment guards against future collapse. |
| T-56-01-D1 | Denial of Service | A pathological template with unbounded `{{...}}` tokens slows rendering | accept | Templates are first-party static assets shipped in-package; token count is bounded and known. |
</threat_model>
