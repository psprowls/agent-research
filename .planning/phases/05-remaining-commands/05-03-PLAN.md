## Phase Goal

**As a** Pat (preparing to ship the `ingest` command), **I want to** port `ingest_source.py` and `ingest_work_item.py` from lattice-wiki-core into vault_io with subprocess `_run_helper()` calls replaced by direct function imports, **so that** the `ingest` command in plan-05-05 can call deterministic extraction + slug + validate helpers as library functions and the work-item write path no longer depends on script-on-PATH assumptions that fail outside the lattice repo layout.

<objective>
Port the two ingest modules from `lattice-wiki-core/` into `cores/vault-io/src/vault_io/` as **library modules only** (no `main()`, no argparse). The critical change is replacing `_run_helper("update_index.py")` / `_run_helper("append_log.py", ...)` subprocess invocations with direct `update_index(wiki)` / `append_log(wiki, ...)` imports (RESEARCH Risk 2). The `_version_check` import is dropped (no vault_io equivalent).

This plan is foundation for plan-05-05's `ingest` vertical slice — it does not directly add a user-visible capability. Justification: porting + command implementation + cross-ref logic exceeds the 2-3-task / ~50% context budget in a single plan. Plan-05-05 builds on this output.

Output:
- `cores/vault-io/src/vault_io/ingest_source.py` (port of `ingest_source.py`, library functions only)
- `cores/vault-io/src/vault_io/ingest_work_item.py` (port of `ingest_work_item.py`, library functions only, with `file_work_item()` extracted from the original `main()` body)
- `cores/vault-io/tests/test_ingest_source.py` (replaces Wave 0 stub)
- `cores/vault-io/tests/test_ingest_work_item.py` (replaces Wave 0 stub)
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem source file → `extract()` | Reads arbitrary path supplied by user; output fed to LLM downstream |
| User-supplied YAML frontmatter → `_parse_frontmatter` | Untrusted YAML; passed through ruamel/yaml-safe parser |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Tampering | `_parse_frontmatter` accepts untrusted YAML | mitigate | Use `yaml.safe_load` (preserved from lattice-wiki-core port — verify no `yaml.load` snuck in) |
| T-05-03-02 | DoS | `extract()` on a 1GB binary file | mitigate | `LARGE_FILE_BYTES = 200 * 1024` constant + `ERROR_FILE_COUNT = 200` cap; preserved from port |
| T-05-03-03 | Information Disclosure | `extract()` reads any path the process can read | accept | Same user context; deep-agents is a local CLI tool |
| T-05-03-04 | Tampering | `file_work_item` overwrites existing pages | mitigate | `force=False` guard; preserved from lattice-wiki-core main() |
</threat_model>
