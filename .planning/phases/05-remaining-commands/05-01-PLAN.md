## Phase Goal

**As a** Pat (single developer running code-wiki-agent locally), **I want to** bootstrap a new vault with `init`, append timestamped log entries with `log`, and configure model/vault paths via a shared `--config` flag (CLI) or `CODE_WIKI_CONFIG` env var (MCP), **so that** the trivial wiki operations work end-to-end through both delivery surfaces and the global config plumbing is in place for the remaining commands.

<objective>
Deliver the two trivial vertical slices of Phase 5 — `log` and `init` — end-to-end through both CLI and MCP, plus the global `--config` plumbing (D-11, D-12, D-13). Resolve the `init_vault.py` Phase 5 TODO by creating `raw/` and `work/` sibling directories. Create all Wave 0 test stub files referenced by VALIDATION.md so downstream plans have failing/skipped tests to flesh out.

Purpose: Establish the shared command-layer pattern (config module + Typer callback + MCP env-var read) and ship the two zero-LLM commands first so that the harder commands (scan, ingest, lint) inherit a working foundation. Per MVP_MODE: after this plan, a real user can run `code-wiki-agent log` and `code-wiki-agent init` end-to-end — capabilities that did not exist before.

Output:
- `code_wiki_agent/config.py` (new) — `WikiConfig` dataclass + `load_config()` + module-level `_active_config`
