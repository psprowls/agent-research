<objective>
Wire `run_scan()` to use graph-io as the source of truth for what gets scanned and where vault pages land. The pipeline gains: (a) a pre-scan `cg update` (incremental, no trace, no model) via Phase 38's `commands/graph.py` helpers; (b) a single read-only graph connection opened after a successful update and closed in `finally`; (c) a decoration step that fetches every workspace's graph URI + domain via one `list_packages` + one domain-join query and stamps `pkg["uri"]` / `pkg["domain"]` onto the discovered dicts; (d) routing-slug recomputation for workspaces whose domain changed; (e) a strict error policy — hard-abort on general cg failure, graceful path-based fallback only on filesystem init-failure with a single prescribed stderr line.

Out of scope: refactoring `discover_workspaces`; modifying `_wiki_relative_path_for`; any change to `packages/wiki-io/`; adding a `--no-graph-update` flag; orphaned-page reconciliation on rename; trace emission from the scanner's cg update.

Output: 1 modified scan command + 1 new unit test file + 1 new integration test file + 1 conftest fixture touch-up.
</objective>

```python
class ScanAbortedError(RuntimeError):
    """Raised when run_scan() must hard-abort because `cg update` failed with
    a non-recoverable runtime error (D-07).

    Carries the cg exit_code and any stderr the cg layer produced so callers
    (CLI / MCP tool) can surface a meaningful diagnostic without re-running.
    """

    def __init__(self, exit_code: int, stderr: str) -> None:
        self.exit_code = exit_code
        self.stderr = stderr
        super().__init__(
            f"cg update failed (exit_code={exit_code}); scan aborted. stderr: {stderr.strip() or '<empty>'}"
        )
```

### 3c. Add module-level constants for the init-failure pattern (D-08)

Below the `ScanAbortedError` definition:

```python
# D-08 init-failure detection: when `cg update` returns GENERIC and stderr matches
# one of these substrings, we treat it as a filesystem init failure (permission/disk)
# rather than a runtime data-correctness failure. Conservative — false positives
# would graceful-fallback on a runtime error; false negatives would hard-abort on
# an unusual init failure. Both are safe-side per Phase 39 RESEARCH §11.
_INIT_FAILURE_STDERR_PATTERNS = (
    "Permission denied",
    "Read-only file system",
    "No space left on device",
    "Errno 13",
    "Errno 28",
    "Errno 30",
)


def _is_init_failure_stderr(stderr: str) -> bool:
    """Return True if `stderr` matches a known init-failure pattern (D-08)."""
    return any(p in stderr for p in _INIT_FAILURE_STDERR_PATTERNS)
```

```python
def _query_package_domains(conn) -> dict[str, str]:
    """Return {package_name: domain_name} for every package with a
    `belongs_to_domain` edge.

    Single SQL round trip. Packages with no domain edge are absent from the
    returned dict (caller uses dict.get with the filesystem-derived default).
    """
    rows = conn.execute(
        "SELECT p.name, d.name FROM nodes p "
        "JOIN edges e ON e.src = p.id AND e.kind='belongs_to_domain' "
        "JOIN nodes d ON e.dst = d.id "
        "WHERE p.kind='package' AND d.kind='domain'"
    ).fetchall()
    return {row[0]: row[1] for row in rows}
```
