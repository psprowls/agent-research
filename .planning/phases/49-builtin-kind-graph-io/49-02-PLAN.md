Catches `FileNotFoundError`, `subprocess.TimeoutExpired` → returns `None`. On non-zero exit → `None`. (b) Cache file = `cache_dir / f"node-builtins-{major}.json"`. If exists, attempt `json.loads(cache_file.read_text())`; on success return `frozenset(...)`. (c) On cache miss / read failure, call `_harvest_node_builtins()`; if `None` → return `frozenset()`. (d) `cache_dir.mkdir(parents=True, exist_ok=True)`, then `cache_file.write_text(json.dumps(harvested))` inside a `try/except OSError` (best-effort write — never raise). (e) Return `frozenset(harvested)`.

    8. **Per-package scanner.** Module-private `_scan_package(repo_root: Path, pkg_name: str, pkg_rel: str | None, file_rels: list[str], node_builtins: frozenset[str]) -> dict[tuple[str, str], set[str]]`. Returns an accumulator keyed by `(language, module_name)` with values as `set[str]` of imported symbols. For each file in `file_rels`:

       - If suffix `.py`: iterate `_PYTHON_IMPORT_RE.finditer(content)`. For each match, take the captured module-spec, split on `.`, top-level segment is the candidate. If `_is_python_stdlib(top)` → add to accumulator under `("python", top)`. To populate `imported_symbols` (D-08), use a second regex pass on the same line as the match to capture named imports — see "Symbol capture" below. For `import X` (no `from`), no symbols are captured (the whole module is "imported"). For JS `import {a, b} from "X"`, capture the braced clause; for `require('X')` or `import 'X'` (side-effect or default), capture nothing. Accept that paren-wrapped multi-line Python imports may capture only the first-line symbols (Assumption A2 in RESEARCH.md — deferred to a future AST migration).

| `update.run` transaction → builtins.refresh | builtins.refresh shares the existing `store.transaction(conn)`; does NOT open a new transaction. Failure modes propagate through the outer transaction's rollback semantics. |
