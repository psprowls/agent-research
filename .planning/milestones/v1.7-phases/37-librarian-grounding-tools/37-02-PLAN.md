       def _estimate_tool_schema_tokens(tools: list) -> int:
           """Approximate the token cost of bound-tool schemas.

           Serializes each tool's name + description + args_schema into a single
           text blob and counts via Bedrock CountTokens (claude-3-5-haiku is the
           default counting model because Claude 4.x models don't support the
           CountTokens operation — see packages/wiki-io/src/wiki_io/update_tokens.py).
           """
           if not tools:
               return 0
           parts: list[str] = []
           for t in tools:
               parts.append(getattr(t, "name", "") or "")
               parts.append(getattr(t, "description", "") or "")
               schema = getattr(t, "args_schema", None)
               if schema is not None:
                   try:
                       parts.append(schema.schema_json())
                   except Exception:
                       parts.append(str(schema))
           blob = "\n".join(parts)
           try:
               return count_tokens(blob)
           except Exception:
               # CountTokens API errored — return 0 so the gate is permissive on
               # API failure (we still abort on the higher-level budget compare
               # if the input itself is already over budget). RESEARCH.md §2.
               return 0

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM → tool-call dispatch | LLM-supplied tool-call `name` + `args` cross into `tool.invoke(args)`. Unknown `name` returns an error string; bad `args` are rejected by the @tool's `args_schema` (LangChain validates before the tool body runs). |
| `run_query` → Bedrock CountTokens API | The pre-flight gate sends `LIBRARIAN_SYSTEM + page text + tool schemas` to Bedrock. The text crosses the AWS boundary. |
| Filesystem (`code.db`) → process | Read-only SQLite over the user's own graph DB. `mode=ro` rejects writes at the SQLite level. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-37-06 | Tampering | LLM emitting a tool_call `name` that doesn't match any bound tool | mitigate | `next((t for t in graph_tools if t.name == call_name), None)` returns None → tool_output is `f"ERROR: unknown tool {call_name!r}"`. Loop continues; LLM can recover. |
| T-37-07 | Information Disclosure | Sending wiki page text + system prompt to Bedrock CountTokens API for budget estimation | accept | The wiki text is already sent to Bedrock during the librarian fan-out itself. CountTokens adds no new disclosure surface. |
| T-37-08 | Denial of Service | LLM enters infinite tool-call loop (always returns tool_calls) | mitigate | `_LIBRARIAN_MAX_ITERS = 5` cap; on cap, return `NO_RELEVANT_CONTENT` and exit drill cleanly. |
