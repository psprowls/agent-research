## Milestone Goal

Land the full ontology spec inside `graph-io` — schema v2, URI identity, all new node and edge types, additive scanner extensions, brand sweep — so v1.7 can integrate graph-io into `graph-wiki-agent` and redesign the wiki on top of it. `graph-io`-only milestone; plugin and existing wiki scripts stay functional and untouched.

## Key Accomplishments

- **Schema v2 + URI identity foundation** — Phase 28 bumped `SCHEMA_VERSION` to 2 (full rebuild required on upgrade), added a dedicated `uri TEXT` column on `nodes` with `idx_nodes_uri`, wired `SCHEMA_MISMATCH` (exit code 4) for v1→v2 mismatches on incremental updates, and shipped `graph_io.uri` with 7 composition helpers (`repo_uri`, `pkg_uri`, `subpkg_uri`, `file_uri`, `domain_uri`, `entry_point_uri`, `test_suite_uri`). `cg update --full` v1→v2 rebuild lands; idempotency test passes.

- **Structural containment tree** — Phase 29 emits one `Repository` node per repo (URL/owner from `git remote`), `SubPackage` nodes for every Python `__init__.py`-containing subdirectory, and `File` nodes with seven role-flag attrs (`is_importable`, `is_executable`, `has_main`, `is_test`, `is_config`, `is_generated`, `is_type_only`). `physically_contains` edges form a strict tree (Repository → Package → [SubPackage* → File | File]); `resolve.sweep` excludes the new structural kinds from path-NULL deletion. `source-parser` extended with `_has_main_block` and `_has_importable_symbols` file-level attrs (SPARSER-01/02). `sample_monorepo` fixture locks STRUCT-04 invariant.

- **EntryPoints + TestSuites** — Phase 30 ships `EntryPoint` nodes from `pyproject.toml [project.scripts]` / `package.json` `bin`/`main`/`exports` with polymorphic `implemented_by` edges (File for path-only entries, Function/Class for callable-syntax entries like `pkg.cli:main`). `TestSuite` nodes derived from FS layout + framework config (pytest/jest/vitest/mocha variants); test files re-parented from Package to TestSuite containment; flat — no suite nesting. Suite-level `tests` edges connect to `Package`, `Domain`, or `Repository`. Call-order pitfall (packages.refresh before test_suites.emit) locked by a fixture test.

- **Domain layer + derived edges** — Phase 31 reads `domains.yaml` at repo root into `Domain` nodes, builds `belongs_to_domain` (multi-domain + zero-domain supported) and `domain_contains_domain` (tree) edges with cycle detection that surgically skips cycle-participating edges and warns. Missing `domains.yaml` is not an error. Derived `references` (Domain → Package with usage count) and `depends_on` (Domain → Domain) edges computed post-update; re-runnable; second `cg update` does not duplicate.

- **Query layer extensions** — Phase 32 shipped 4 frozen dataclasses (`RepositoryDescription`, `DomainDescription`, `EntryPointDescription`, `SuiteDescription`), 4 `describe_*` + 6 `list_*` helpers, plus a recursive `_DOMAIN_DESCENDANTS_CTE` powering cross-cutting and bubble-up queries (`tests_for_package`, `entry_points_for_package`, `tests_for_domain`, `domain_references`, `domain_depends_on`, `cross_cutting_packages`). `describe_package` extended with `domains:` / `entry_points:` / `test_suites:`; `describe_path` extended with `role_flags:`. `_VALID_KINDS` allow-list backs `cg find`. 29 Wave 1 + 15 Wave 2 unit tests, including `test_cte_cycle_safe` with a `signal.alarm` guard.

- **CLI surface** — Phase 33 added `exit_codes.AMBIGUOUS = 7` (deviation: STALE=2 already taken), extended `cg status` with a `repository:` first line, and shipped 12 new `q_*` modules registering 14 new subcommands: `describe-repo`, `list-packages`, `list-entry-points` (with `--kind`), `list-scripts` (with annotation + dedup), `list-suites`, `describe-suite`, `what-tests` (probe-both for `--kind=package|domain`), `list-domains`, `describe-domain` (nested sub-blocks via inline SQL), `domain-refs`, `domain-deps`, `cross-cutting`. `cg --help` lists 25 subcommands (13 pre-existing + 12 new). `test_cli_anti_regression.py` pins all 13 pre-existing subcommands.

- **Brand sweep — no backwards compat** — Phase 34 replaced every `lattice`/`LATTICE` reference in `packages/graph-io/` (README first-line title, tagline, `~/.lattice/graph/code.db` → `paths.graph_dir(workspace)`, `plugins/lattice-graph/` → `plugins/graph-wiki/`, CLI description, four test files). The deprecated `LATTICE_GRAPH_LOCK_TIMEOUT_MS` env var alias-with-warning was **revised mid-milestone** (operator decision: single-user repo, no backwards compat) to a straight rename to `GRAPH_WIKI_LOCK_TIMEOUT_MS`. The dead `_SKIP_REPO_PREFIXES = ("lattice/",)` filter and its test were deleted (the target `lattice/` vendor directory does not exist in this repo). `.brand-grep-allow` shipped with broader-codebase carve-outs only; **zero `packages/graph-io/` entries**. SC#3 collapsed from three manual scenarios to a single automated grep + pytest gate.
