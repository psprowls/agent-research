<objective>
Add consecutive-same-role group collapsing to the `code-wiki-agent trace` renderer with a `--expand` flag that reverts to today's one-line-per-record behavior. This is OBS-06.

Purpose: Per D-11 (group = maximal run of ≥2 records where `role` matches AND record has NO `event` key AND no `kind` key), D-12 (threshold N=2), D-13 (dense single-line group summary shape with ISO-8601 timestamps, status breakdown, summed tokens/cost, optional `(+K unknown)`), and D-14 (single boolean `--expand` flag — no per-role or per-threshold variants in v1.1).

Output: Renderer detects consecutive same-role groups, emits one dense summary line per group; non-groupable records (`event`, `kind`) and isolated records render full-line. `--expand` flag disables collapsing entirely. Four syrupy snapshot tests lock the four key shapes.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User-supplied trace file → renderer | Trace records are read from a path the user supplied to `trace <file>`. The collapse-pass walks records sequentially; malformed records must not crash the walk. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-09 | Denial of Service | Group-detection pass over very large trace files | accept | Implementation is O(N) with a sliding window of size ≤ run length; no quadratic blowup. Typer/Python stdout buffering handles large output. Trace files in practice are bounded by fan-out size (~10s of records). |
| T-09-10 | Tampering | A record with control characters in `role` could corrupt the collapsed line | mitigate | `typer.echo` writes raw strings; if a malicious `role` value contains ANSI escapes, terminal could be confused. Mitigation: the `role` value originates from internal config (models.toml roles), not from user input — same trust boundary as `_render_trace_record` today. Acceptance: do NOT add explicit escaping in this plan (out of scope); document in `docs/trace-schema.md` that `role` is producer-controlled config. Risk accepted. |
| T-09-11 | Information Disclosure | `--expand` mode reveals every record including potentially long `error` strings | accept | Same surface as today's renderer; no new exposure. Plan 09-02 documents that errors are truncated/preserved as the producer writes them. |
</threat_model>
