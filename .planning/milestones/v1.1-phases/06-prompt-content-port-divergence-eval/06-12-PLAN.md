<objective>
Close UAT gap G1 (ingestor LLM emits markdown-code-fenced frontmatter ` ```yaml ... ``` ` instead of starting with `---`, causing ING-001 to fail). Apply two defenses: (a) tighten the ingestor system prompt with an explicit no-fence rule; (b) make `_parse_ingestor_response` strip a leading and trailing code fence around the YAML block before delimiter detection.

Purpose: ING-001 (`text.startswith('---')`) is a hard divergence-check rule, and the live UAT showed the LLM violating it. Per the anti-shallow rule, both prompt and parser must be fixed — we cannot rely solely on the LLM following a new instruction.

Output: Updated `prompts/ingestor.py` (one new private rule constant, no semantic change to other fragments), updated `commands/ingest.py:_parse_ingestor_response` (peels code fence pre-parse), and unit tests in `tests/unit/test_commands_ingest.py` that fail without the parser fix and pass with it.
</objective>

    1. Modify `_parse_ingestor_response(text: str)`. At the top of the function, after `text = text.strip()` (line 111), add a fence-stripping preamble:

       ```
       # Defense-in-depth: ingestor LLM may wrap the frontmatter in a markdown
       # code fence (```yaml ... ``` or ``` ... ```). The system prompt forbids
       # this (prompts/ingestor.py:_NO_CODE_FENCE), but we strip defensively so
       # ING-001 (startswith '---') passes even on prompt-rule violations.
       if text.startswith("```"):
           # Strip opening fence line (```yaml or just ```)
           nl = text.find("\n")
           if nl == -1:
               return {}, text
           text = text[nl + 1 :].lstrip("\n")
           # Strip trailing fence — find the LAST occurrence of a line that is
           # exactly ``` (possibly followed by trailing whitespace then EOF).
           # Use rsplit to peel from the end without disturbing inline backticks.
           lines = text.rstrip().split("\n")
           if lines and lines[-1].strip() == "```":
               text = "\n".join(lines[:-1]).rstrip()
       ```
