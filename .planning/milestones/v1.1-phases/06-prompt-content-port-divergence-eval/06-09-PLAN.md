    Implement methods:

    `run_programmatic(self, outputs: list[tuple[str, AgentOutputProxy]]) -> dict[str, dict]`:
      Initialize `results = {c.id: {"runs": 0, "failures": 0, "accepted_failures": []} for c in self.checks}`.
      For each (fixture_id, output) in outputs:
        For each check in self.checks:
          `results[check.id]["runs"] += 1`
          `passed, excerpt = _run_check_one(check, output, self.vault)`
          if not passed:
            `results[check.id]["failures"] += 1`
            `results[check.id]["accepted_failures"].append({"fixture": fixture_id, "excerpt": excerpt[:200]})`  # cap excerpts at 200 chars
      Return `results`.

    `run_judge(self, outputs: list[tuple[str, AgentOutputProxy, str]]) -> dict[str, dict]`:
      Lazy-import `from deepeval.metrics import GEval; from deepeval.test_case import LLMTestCase, SingleTurnParams`.
      The rubric markdown distills judge-only rules (LIB-005, LIB-006, etc.). For accumulation purposes, the metric uses a single per-role judge rule id `f"{ROLE_PREFIX}-JUDGE"` where ROLE_PREFIX is LIB/ING/LNT/SCN. (Specific sub-IDs LIB-005, LIB-006 from RESEARCH inventory are encoded in the rubric text as criteria, not separated by metric.) Document this in the method docstring: "Judge pass aggregates rubric-level pass/fail per fixture; granular judge-only IDs (LIB-005, LIB-006, ING-005, ING-006, LNT-004, LNT-005, SCN-005) are encoded as criteria within the rubric text and surface in the accepted_failures excerpt as the judge's reason string."
      Initialize `judge_id = f"{self.role.upper()[:3]}-JUDGE"` (e.g., "LIB-JUDGE", "ING-JUDGE", etc — for linter use "LNT-JUDGE", for scanner use "SCN-JUDGE"). Use an explicit mapping `{"librarian": "LIB-JUDGE", "ingestor": "ING-JUDGE", "linter": "LNT-JUDGE", "scanner": "SCN-JUDGE"}` to avoid ambiguity.
      Results: `{judge_id: {"runs": 0, "failures": 0, "accepted_failures": []}}`.
      For each (fixture_id, output, query) in outputs:
        scores = []
        reasons = []
        for cfg in JUDGE_PANEL_CONFIG:
          `judge = make_judge(cfg)`  # fresh per call per RESEARCH §Pitfall
          `metric = GEval(name=f"divergence_{self.role}", criteria=self._rubric_text, evaluation_params=[SingleTurnParams.INPUT, SingleTurnParams.ACTUAL_OUTPUT], model=judge, threshold=_JUDGE_THRESHOLD)`  # `model=judge` is non-negotiable — T-06-04 invariant
          `tc = LLMTestCase(input=query, actual_output=output.answer)`
          `metric.measure(tc)`
          `scores.append(metric.score); reasons.append(metric.reason or "")`
        `mean_score = sum(scores) / len(scores)`
        `results[judge_id]["runs"] += 1`
        if mean_score < _JUDGE_THRESHOLD:
          `results[judge_id]["failures"] += 1`
          `results[judge_id]["accepted_failures"].append({"fixture": fixture_id, "excerpt": (reasons[0] or "")[:200]})`
      Return `results`.

    `run(self, outputs: list[tuple[str, AgentOutputProxy, str]]) -> dict[str, dict]`:
      Programmatic outputs need only (fixture_id, output) — strip the query: `prog_outputs = [(fid, op) for (fid, op, _) in outputs]`.
      `prog_results = self.run_programmatic(prog_outputs)`
      `judge_results = self.run_judge(outputs)`
      Merge: `return {**prog_results, **judge_results}` (keys do not collide: programmatic uses LIB-001..LIB-004, judge uses LIB-JUDGE).
