<objective>
Wire the four new shared fragments (`ARCHITECTURE_OVERVIEW`, `STYLE_RULES`, `LOG_FORMAT`, `CLAUDE_MD_DISAMBIGUATION`) into the four prompt builders (`scanner.py`, `linter.py`, `ingestor.py`, `librarian.py`). Convert each module's prompt assembly from a module-level constant into a `build_X_system(project_context: str = "") -> str` function while preserving the existing module-level constants as backward-compatibility aliases (built with `project_context=""`).

Purpose: This is the wiring half of CTX-01 and the prompt-side half of CTX-03. After this plan, every prompt builder accepts an optional `project_context` and composes the right curated fragments per the CONTEXT.md §Wiring matrix. After plan 10-06 ships the command-side wiring, the agents actually receive the new context.

Output: Four modified prompt files. The existing snapshot tests (`test_*_system_snapshot` in test_prompt_snapshots.py) will FAIL until snapshots are regenerated — that regeneration is intentional and is part of the verification step here.

Per CONTEXT.md §Wiring matrix:
- scanner: project_context + ARCHITECTURE_OVERVIEW + (existing fragments) + LOG_FORMAT
- linter (all 3 groups): project_context + (existing fragments) + LOG_FORMAT + CLAUDE_MD_DISAMBIGUATION
- ingestor: project_context + ARCHITECTURE_OVERVIEW + STYLE_RULES + CLAUDE_MD_DISAMBIGUATION + LOG_FORMAT + (existing fragments, _NO_CODE_FENCE last)
- librarian: STYLE_RULES only; no project_context wiring
</objective>
