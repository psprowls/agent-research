  <action>
Replace the `NotImplementedError` body of `compute_clusters` with a full implementation. Add private helpers used by the public function. Required helper functions (D-23):

- `_load_package_names(conn) -> list[str]` — `SELECT name FROM nodes WHERE kind = 'package' ORDER BY name`. Returns alphabetical list. Sorted at the SQL level for determinism.

- `_load_reference_edges(conn) -> list[tuple[str, str]]` — runs the D-15 query exactly:
  ```sql
  SELECT src.name, dst.name
  FROM edges e
  JOIN nodes src ON src.id = e.src
  JOIN nodes dst ON dst.id = e.dst
  WHERE e.kind = ? AND src.kind = ? AND dst.kind = ?
  ORDER BY src.name, dst.name
  ```
  Parameters: (`_REFERENCES_KIND`, `_PACKAGE_KIND`, `_PACKAGE_KIND`). Returns sorted list (sorted at SQL level for determinism).

- `_compute_hubs(adjacency_in: dict[str, set[str]], names: list[str], threshold: float) -> set[str]` — returns the set of names that are hubs. For each name n, `in_degree = len(adjacency_in[n])`. If `n_packages <= 1`, return empty set (no denominator). Else `is_hub = in_degree / (n_packages - 1) > threshold` (strictly greater).

- `_compute_intra_in_degree(members: set[str], original_edges: list[tuple[str, str]]) -> dict[str, int]` — returns `{member: count_of_edges_dst==member_AND_src_in_members}`. Used by D-10 cluster naming.

- `_pick_cluster_name(members_sorted: list[str], intra_in_degree: dict[str, int]) -> str` — pick member with max intra-cluster in-degree; tiebreak alphabetical. D-10.

- `_compute_connects_clusters(hub: str, original_edges: list[tuple[str, str]], name_to_cluster_id: dict[str, int]) -> tuple[int, ...]` — for every edge touching `hub`, take the other endpoint; if it appears in `name_to_cluster_id`, add that cluster id. Dedup. Sort ascending. Return tuple. Hub-to-hub edges (other endpoint not in any cluster) are ignored.

- `_detect_degenerate(clusters: tuple[Cluster, ...], n_packages_total: int, threshold: float) -> str | None` — D-11..D-14: check giant FIRST (max size / total > 0.8 if any clusters exist); else check all-singletons (clusters non-empty AND len(clusters) == n_packages_total); else None. Suggestion thresholds: 0.3 for giant, 0.7 for singletons (D-14). Hard-code the exact warning strings from D-11/D-12 verbatim. Use Python's `:.0%` formatter for the giant percentage and `repr(threshold)`-style for the threshold value (use `f"{threshold:g}"` for a clean float display, e.g., 0.5 not 0.5000000001).

Body of `compute_clusters`:
1. Validate `hub_threshold` is in `(0.0, 1.0]`. If not, `raise ValueError(f"hub_threshold must be in (0.0, 1.0], got {hub_threshold}")` (D-06).
2. `names = _load_package_names(conn)`. `n_packages_total = len(names)`.
3. Early return if `n_packages_total == 0`: empty ClusterResult with `hub_threshold`, `n_packages_total=0`, `degenerate_warning=None`, empty tuples (D-22).
4. `edges = _load_reference_edges(conn)`. Build `adjacency_in: dict[str, set[str]] = defaultdict(set)` — for each `(src, dst)` add `src` to `adjacency_in[dst]`. Default-zero for names not appearing.
5. `hubs = _compute_hubs(adjacency_in, names, hub_threshold)`.
6. `remaining_names = [n for n in names if n not in hubs]`. `remaining_edges = [(s, d) for s, d in edges if s not in hubs and d not in hubs]`.
7. Initialize `uf = _UnionFind(remaining_names)`. For each `(src, dst)` in `remaining_edges`: `uf.union(src, dst)`. (Iterate over edges in their SQL-sorted order to preserve determinism — union-find result is order-invariant but defensive.)
8. Group remaining_names by `uf.find(n)`. Build a `dict[root, list[str]]`.
9. For each group: sort members alphabetically (D-09 inner sort); compute intra_in_degree using `_compute_intra_in_degree(set(group_members), edges)`; pick name via `_pick_cluster_name`.
10. Build pre-sort cluster tuples (placeholder id 0; will assign after sort). Build a list of dicts `{"name": name, "members_sorted": tuple, "size": int}`.
11. Apply outer sort key=lambda c: (-c["size"], c["members_sorted"][0]) (D-09). Assign ids 0,1,2,... post-sort.
12. Build `name_to_cluster_id: dict[str, int]` mapping every member to its cluster id.
13. Construct final `Cluster(id=i, name=..., members=tuple, size=...)` instances. Hold in a tuple in the sorted order.
14. For each hub h (alphabetical order — sorted(hubs)): compute `imported_by_count = len(adjacency_in[h])`; `imported_by_fraction = imported_by_count / (n_packages_total - 1)` if `n_packages_total > 1` else 0.0; `connects_clusters = _compute_connects_clusters(h, edges, name_to_cluster_id)`. Build `CrossCuttingHub` tuples. Already alphabetical from sorted(hubs) iteration — final cross_cutting tuple is naturally name-sorted (D-09).
15. `degenerate_warning = _detect_degenerate(clusters_tuple, n_packages_total, hub_threshold)`.
16. Return `ClusterResult(hub_threshold=hub_threshold, n_packages_total=n_packages_total, degenerate_warning=degenerate_warning, clusters=clusters_tuple, cross_cutting=cross_cutting_tuple)`.

The function must not write to the DB. The function must not call `print()`. All side-channel output (degenerate warning) flows through `ClusterResult.degenerate_warning`.

Concrete strings for degenerate_warning (D-11/D-12 verbatim — must match the test assertions exactly):
- Giant: `f"warning: domain clustering degenerate — cluster '{cluster_name}' contains {pct:.0%} of packages.\nLikely cause: hub threshold too high (currently {threshold:g}), too few packages, or sparse imports.\nTry: cg domain-clusters --hub-threshold 0.3"`
- All-singletons: `f"warning: domain clustering degenerate — every package is its own cluster.\nLikely cause: hub threshold too aggressive (currently {threshold:g}) or no inter-package imports.\nTry: cg domain-clusters --hub-threshold 0.7"`
