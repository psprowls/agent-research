<decisions>
## Implementation Decisions

### Rename Scope (D-01)
- **Verb chosen:** `bootstrap` across all three user-facing surfaces. Picked over `init-wiki` (clunky duplication), `wiki_init_wiki` (awkward triple-wiki), `scaffold` (CLI tooling convention but less semantically precise), `setup` (too casual), `new` (ambiguous).
- **Rationale:** Reads as a deliberate action ("bootstrap a wiki from scratch"), avoids the `wiki_init_wiki` triple-noun problem, and creates uniform vocabulary: `/graph-wiki:bootstrap` / `code-wiki-agent bootstrap` / `wiki_bootstrap`.

### Surface Coverage (D-02)
- **All three user-facing surfaces rename:** slash command, Bedrock CLI subcommand, MCP tool — for vocabulary consistency. User specifically chose "Also rename CLI + MCP for consistency" over "slash command only".
- **Machine-facing internals stay:** `init_vault.py` script + `init_vault()` function untouched. They're called by the renamed surfaces (slash command body / CLI / MCP) but not user-typed. Avoids churning workspace imports for no user-facing benefit.

### Historical Sweep (D-03)
- **All 19 `.planning/` references swept** (archived phase docs, milestone docs, prior CONTEXT/PLAN/VERIFICATION files, RETROSPECTIVE.md). Matches Phase 21's `code-wiki-agent` rename norm — full historical consistency, future grep for `/graph-wiki:init` returns 0.
- **Acknowledged trade-off:** alters historical record vs. git log/commit messages (which retain the old name immutably). Same precedent set by Phase 21 D-05.

### No Backwards-Compat (D-04)
- **Hard cut, no shim.** No stub `init.md` redirector to `/graph-wiki:bootstrap`. No Typer alias mapping `code-wiki-agent init` to the new function. No MCP tool registered under both `wiki_init` and `wiki_bootstrap`.
- **Rationale:** Aligns with CLAUDE.md "Avoid backwards-compatibility hacks" + Phase 20 precedent (hard-deleted `WikiConfig.models_path` / `--config` / `CODE_WIKI_CONFIG`). User reinstalls plugin / re-runs `uv sync`.

### Brand-Gate Enforcement (D-05)
- **Extend `scripts/check-brand.sh`** to grep for the OLD slugs and fail CI on reintroduction:
  - `graph-wiki:init\b` (word boundary so the rename target `graph-wiki:bootstrap` passes cleanly)
  - `wiki_init\b` (catches the old MCP tool name; the new `wiki_bootstrap` passes)
  - `def init(` and `@app.command()\ndef init` patterns in `agents/code-wiki-agent/src/code_wiki_agent/cli.py` (catches accidental reintroduction of the Typer subcommand)
- **`.brand-grep-allow` updated** for narrow exemptions only (e.g., this CONTEXT.md and the corresponding DISCUSSION-LOG.md retain the old names in quoted documentation of the rename — those entries get allowlisted by exact path).
- **Decision:** land in this phase (not deferred to Phase 21). The brand-gate is small enough that bundling it here is cleaner than adding cross-phase coupling.

### Cutover Strategy (D-06)
- **Staged commits per layer**, each landing green:
  1. Rename the slash command file (`plugins/graph-wiki/commands/init.md` → `bootstrap.md`) with `git mv` to preserve history; update body text inside
  2. Rename the MCP tool (`server.py`: `wiki_init` → `wiki_bootstrap`; `WikiInitInput`/`WikiInitOutput` → `WikiBootstrapInput`/`WikiBootstrapOutput`); update unit tests
  3. Rename the Typer subcommand (`cli.py`: `def init` → `def bootstrap`); update help text and unit tests
  4. Sweep the 11 active-source consumer references (plugin docs, agent docs, vault-io consumers)
  5. Sweep the 19 historical `.planning/` references
  6. Extend `scripts/check-brand.sh` + `.brand-grep-allow`; verify gate runs green
- **Per-commit gate:** `uv run --package code-wiki-agent pytest agents/code-wiki-agent/tests/ -m "not integration"` must pass before each commit lands. Integration tests gated by `CODE_WIKI_RUN_INTEGRATION` are not required per intermediate commit.

### Verification Approach (D-07)
- **SC#1:** `ls plugins/graph-wiki/commands/init.md` exits non-zero AND `ls plugins/graph-wiki/commands/bootstrap.md` exits zero
- **SC#2 (re-mapped to bootstrap surface):**
  - `grep -rln 'graph-wiki:init\b' plugins/ packages/ agents/ scripts/ docs/ README.md CLAUDE.md` returns 0
  - `grep -rln '\bwiki_init\b' agents/` returns 0
  - `grep -rn 'def init\b' agents/code-wiki-agent/src/code_wiki_agent/cli.py` returns 0
- **SC#3 (Claude Code native `/init` reachable):** manual smoke test — install/reinstall the plugin, type `/init` in Claude Code, confirm the native "initialize CLAUDE.md" workflow fires (not the graph-wiki bootstrap workflow)
- **Additional checks added in planning:**
  - `code-wiki-agent --help` lists `bootstrap`, does not list `init`
  - `python -c "from code_wiki_mcp import server; print(server.wiki_bootstrap)"` succeeds; `wiki_init` raises AttributeError
  - `bash scripts/check-brand.sh` exits 0 on a clean tree; intentionally injecting `/graph-wiki:init` into a random file causes it to exit non-zero (red-then-green sanity)

### Folded Todos
- `.planning/todos/pending/2026-05-19-rename-graph-wiki-init-command-to-init-wiki.md` — folded into this phase. The todo name references the older "init-wiki" direction; the actual rename target is now `bootstrap`. Todo file gets moved to `.planning/todos/resolved/` after the phase ships.
