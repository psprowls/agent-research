    1. Raise `ValueError("short_filename: empty uri")` if `not uri`.
    2. Split uri once on `:` with `maxsplit=1`. If no `:`, raise `ValueError(f"short_filename: malformed uri {uri!r}: missing `:` prefix separator")`. Let `uri_prefix` = first part, `path` = second part.
    3. Map `uri_prefix` to the filename-layer prefix using a NEW module-level dict `_FILENAME_PREFIX_BY_URI_PREFIX = {"repo": "repo", "pkg": "pkg", "app": "app", "domain": "domain", "plugin": "plugin", "dependency": "dep", "test_suite": <derived>}`. Place this dict immediately above the `short_filename` def. The `dependency → dep` mapping here is the filename-layer alias (D-05). For `test_suite`, the prefix is derived from `suite_kind` (see step 4) so the value in this dict is unused for that kind — keep an explicit entry of `"test_suite"` as the dict key with value `"tests"` to keep the dict total and serve as the suite_kind=None fallback prefix; the test_suite branch in step 4 overrides as needed.
    4. Compute the plain stem:
       - If `uri_prefix == "test_suite"`: derive `kind_prefix` from `suite_kind`: `unit` → `unit_tests`; `integration` → `int_tests`; `e2e` → `e2e_tests`; `contract` → `contract_tests`; any other value or `None` → `tests`. Then derive `pkg_part`: `pkg_for_suite` if provided, else parse `path.split("/")` and take segment `[-2]` if `len(segments) >= 2`, else segment `[-1]`. plain_stem = `f"{kind_prefix}_{pkg_part}"`.

    5. If `uri in collision_set`: compute `suffix = hashlib.sha256(uri.encode("utf-8")).hexdigest()[:6]` and return `f"{plain_stem}__{suffix}"`. Else return `plain_stem`.

    Write a clear NumPy-style docstring referencing D-03, D-04, D-07, the derivation rules table, and example outputs. ## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Graph URI → short_filename | The graph URI is generated by `graph_io.uri` builders which validate org/repo/name fragments; `short_filename` accepts whatever string the caller passes. | short_filename return → filesystem path | The return value is concatenated with `.md` and appended to `entities_dir` by the caller (Plan 02). Pathlib normalization in the caller is unchanged; this plan does not modify path-joining behavior. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-52-01-T | Tampering | `short_filename` pure helper | mitigate | Pure function with no I/O, no SQL, no global state writes. Cannot be tampered with via external state. Property tests verify deterministic input → output mapping. |
| T-52-01-V | Input Validation | `short_filename` uri parameter | mitigate | Three explicit `ValueError`s gate against empty uri, missing `:` separator, and unknown URI prefix. Negative test cases asserted in `test_value_errors`. |
| T-52-01-I | Information Disclosure | sha256 suffix derivation | accept | sha256 of the full URI is computed but never logged; the 6-hex output is a public-by-design vault filename component. No secret material is hashed. |
| T-52-01-D | Denial of Service | sha256 + dict lookup cost per call | accept | sha256 of a sub-200-byte URI is well under 1µs; called O(N_entities) per `write_entities` invocation; negligible vs. existing per-entity file I/O. |
| T-52-01-R | Repudiation | filename derivation lineage | accept | Filename is deterministic from `(uri, collision_set, suite_kind, pkg_for_suite)`; reproducible by reading the URI + scanning the graph. No audit log required. |
| T-52-01-S | Spoofing | none applicable | accept | Pure function with no identity layer. |
</threat_model>
