<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → handlers | args.name is a user-provided string; bound as a sqlite3 ? parameter inside describe_app — no string concat into SQL |
| store.read_only_connect → query layer | Connection is read-only; even an injected name cannot mutate data |
| subprocess.run(cg, ...) in tests | Inputs are pytest-controlled tmp_path manifests; no untrusted I/O |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-50-03-T | Tampering | describe_app SQL | mitigate | All filter values bound via ? parameters (mirrors describe_package and the audited describe_builtin pattern from Phase 49 Plan 03) |
| T-50-03-I | Information Disclosure | list_apps + describe_app exposing app_signals | accept | Repo source is the trust boundary; app_signals records public manifest content |
| T-50-03-D | Denial of Service | describe_app multi-JOIN consumer query | accept | Bounded by node count in the local graph; same complexity profile as describe_package |
| T-50-03-V | Input Validation | q_describe_app argparse name positional | mitigate | argparse enforces presence; downstream describe_app handles arbitrary strings safely via parameterized SQL |
</threat_model>
