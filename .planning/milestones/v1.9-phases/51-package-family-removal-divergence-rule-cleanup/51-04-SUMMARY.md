1. **Tokenizer source = production** — the plan's example used a generic `bm25s.Tokenizer(stopwords="english", stemmer=None)`. The fixture's original BM25 files were written by `build_index()`, so matching production tokenization is the only way to keep the fixture consistent on future regens.
2. **`search.db` untouched** — per RESEARCH.md Pitfall 5 / Assumption A3. BM25-only regen has zero Bedrock dependency; the embedded vectors are stale relative to the new corpus but no test reads them for content correctness (verified in RESEARCH.md A3).

Two such deletions: the H2 "Why `package-family` is genuinely new vocabulary" section (+ its "Applies to" enumeration and "Service families" deferral note) in `lattice-dependencies-tiering.md`, and the dedicated `package-family` row in the `lattice-vault-terminology.md` glossary. ## Deviations from Plan

### 1. [Rule 1 — Tooling correction] Used production `_build_tokenizer` instead of the plan's example tokenizer

- **Found during:** Task 2 (BM25 regen script writing).
- **Issue:** The plan's `<action>` block specified `tokenizer = bm25s.Tokenizer(stopwords="english", stemmer=None)`. That would have produced a vocab incompatible with the production `build_index()` tokenizer, which uses a specific regex splitter (`[a-zA-Z0-9][a-zA-Z0-9_\-']+`), `lower=True`, and a ~60-word stopword list (copied verbatim from lattice-wiki-core).
- **Fix:** Imported `_build_tokenizer` and `_discover_pages` from `graph_wiki_agent.commands.query` directly, so the regen call sequence becomes identical to lines 758-769 of `build_index()`. RESEARCH.md Pitfall 5 actually anticipated this ("extract lines 753-769 of `query.py` into a one-shot script") — the plan's example was a simplification.
- **Files modified:** none (the regen script was a one-shot at `/tmp/regen_bm25.py`, not committed).
- **Verification:** `test_round_trip` passes; the vocab files written are structurally identical (same params.index.json, same file count) to what was previously committed by the lattice-wiki-core pipeline.
- **Committed in:** `ac1c111` (Task 2 commit — the deviation is in the *method* used to produce the artifacts, not in the artifacts themselves).
