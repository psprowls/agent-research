# v1.8 Scope Candidate — Wiki Entity Restructure

## Trigger

Surface this when `/gsd-new-milestone` is run to bootstrap v1.8. The previous v1.8 (URI-Keyed Wiki & Reconciliation) was aborted on 2026-05-26 before research synthesis landed. The user is reconsidering the milestone direction; this restructure proposal is the leading candidate.

## One-line summary

Collapse the wiki's parallel page-type-per-directory layout into a single unified entity model driven by the graph, with edges as scanner-populated frontmatter relations and a domain-first index that interleaves graph-derived entities with the existing curated lanes (concepts, adrs, architecture, work, sources).

## Why it might be v1.8

- v1.6 built the ontology, v1.7 wired it into the agent — v1.8 was always intended to surface the graph to wiki users. The original URI-keying scope was one *mechanism* for that surfacing; this entity restructure is a more ambitious framing of the same goal.
- URI-keying isn't lost — it becomes one sub-problem (entity keying, Q1 in research questions) inside the larger restructure rather than the headline.
- The user explicitly said "everything is fair game for optimization and improvement" after the abort, and existing exploratory vaults are disposable, so the format-compatibility constraint is relaxed for this milestone.

## Why it might NOT be v1.8

- Scope is large: schema migration + scanner rewrite + template refactor + index regeneration + inbound-link migration. This may be too much for one milestone.
- Could be split: e.g. v1.8 = entity layer + scanner relation-frontmatter (the foundation); v1.9 = index restructure + per-kind template polish.

## Design summary

See `.planning/notes/wiki-entity-restructure-design.md` for the full design captured from the 2026-05-26 explore session. Highlights:

- Two-lane wiki: `/entities/` (graph-derived, flat, `kind`-discriminated) + existing curated lanes unchanged in location.
- Entity kinds admitted at the file boundary: `repository`, `domain`, `package`, `package-family`, `plugin`, `dependency`, `test-suite`. In-file nodes (class, function, method) excluded. New graph kinds need explicit admit/deny.
- Per-kind templates replace directory templates; package-as-folder collapses into single `entity-package` page.
- Graph edges → frontmatter relations, populated by the scanner; prose reserved for narrative.
- Index: domain-first hierarchy, then global by-kind sections. Entities appear in both views.
