Fix the eval-harness so ingestor sweep cells can run in isolation. `EvalWorktree`
currently copies the source wiki into `<tmp>/wiki` but never creates a graph DB,
so the ingestor — which opens the graph via `read_only_connect` and hard-fails
with `IngestorGraphNotInitializedError` when missing (intentional production
behavior, Phase 40 SC#2) — fails on every sweep cell (confirmed: 0 ok / 21 err
across all 7 candidates, 2026-05-29 live sweep).

opens it and proceeds (finding no nodes to slug-align — acceptable for a uniform
cost/format model sweep). There is no source repo behind the round-trip-vault
fixture, so we must NOT run a real `graph_io.update.run(...)` build; we only need
an openable, schema-valid EMPTY DB created via the existing
`graph_io.store.connect(db_path, create=True)` primitive.

Purpose: unblock the ingestor role in eval sweeps without altering the ingestor's
production hard-fail contract.
Output: surgical change to `isolation.py` (+ two imports), a workspace-dep check
in `pyproject.toml`, and an offline test.
</objective>
