<objective>
Predecessor quick `260521-hfr` fixed the `[[wiki/...]]` prefix in package overview
templates, but the container path was hardcoded as `wiki/packages/`. The scanner
agent emits package pages for non-`packages/` containers too — agents
(`wiki/agents/<slug>/`) and plugins (`wiki/plugins/<slug>/`) — which produces
8 broken sub-page wikilinks on the current real wiki:

- `wiki/agents/graph-wiki-agent/graph-wiki-agent.md` emits links to `wiki/packages/graph-wiki-agent/{api,context,patterns,work}` (4 broken — should be `wiki/agents/...`)
- `wiki/plugins/graph-wiki/graph-wiki.md` emits links to `wiki/packages/graph-wiki/{api,context,patterns,work}` (4 broken — should be `wiki/plugins/...`)

Root cause: `package/overview.md` has no template variable to vary the
container segment.

Fix: introduce a `{{CONTAINER_DIR}}` template variable in `package/overview.md`,
document it in the scanner agent's instructions so the LLM substitutes the
right value (`packages` / `agents` / `plugins` / `apps`) based on where the
page is being written, extend the regression test to cover both `agents` and
`plugins` substitution, then re-emit the two affected live-wiki pages and
verify lint reports zero broken links.
</objective>
