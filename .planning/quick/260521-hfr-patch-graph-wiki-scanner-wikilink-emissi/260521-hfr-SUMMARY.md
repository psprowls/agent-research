# Quick 260521-hfr Summary

Patch the two scanner-emission defects responsible for ~41 broken wikilinks on a freshly bootstrapped + scanned wiki: (1) overview / context templates emit unprefixed `[[<container>/...]]` wikilinks that don't resolve against `link_targets`, and (2) `wiki/index.md` always emits `## More` links to five section-index pages but `init_wiki` only seeded `dependencies/index.md`.

## Changes

### Task 1 ‚Äî `fix(260521-hfr)` (`6c98fb8`)

Prefixed every wikilink in the three offending page templates with `wiki/`:

- `package/overview.md` ‚Äî four Sub-pages bullets (`api`, `patterns`, `work`, `context`); column alignment of the trailing `‚Äî` descriptions preserved.
- `package/context.md` ‚Äî all six example wikilinks (Concepts, Decisions, Sources, Belongs to domain, Used by, Related dependencies).
- `domain/overview.md` ‚Äî the HTML-comment example package heading and the `details` Sub-pages bullet.

Test changes:

- `test_package_overview_renders_path_qualified_wikilinks` now asserts the `[[wiki/packages/myslug/<sub>|<sub>]]` form and added negative asserts for the unprefixed regression.
- `test_domain_overview_renders_path_qualified_wikilinks` now asserts `[[wiki/domains/myslug/details|details]]` and added a negative regression assert.
- New test `test_package_context_template_uses_wiki_prefix` reads `package/context.md` directly and verifies every `[[‚Ä¶]]` target starts with `wiki/`.

### Task 2 ‚Äî `feat(260521-hfr)` (`0cbb615`)

`init_vault.init_wiki` now writes stub `index.md` files into `concepts/`, `sources/`, `adrs/`, `architecture/` after the `FIXED_VAULT_DIRS` mkdir loop and before schema-file rendering, with an idempotency guard (`if stub_path.exists(): continue`) so existing user content is never clobbered.

Each stub is `# <Label>\n\nAuto-generated stub. Pages of category `<lowercase-label>` appear here as they are added to the wiki via `/graph-wiki:ingest` or manual creation. Regenerated by `scripts/update_index.py` once entries exist.\n`.

Skipped `dependencies/index.md` because the scanner's `regenerate_dependencies_index` already seeds it with the marker-bounded auto-block.

Added `test_init_wiki_creates_section_index_stubs` covering both the creation path and the re-init idempotency guard (writes a sentinel, re-runs `init_wiki(... force=True)`, asserts the sentinel survives).

### Task 3 ‚Äî End-to-end verification (no source edits)

Copied `/Users/pat/Personal/wikis/deep-agents/` to `/tmp/wiki-verify/`, re-ran `init_vault` with `--force` against the tmp copy, and re-ran lint.
