<objective>
Make `workspace_io.init.init()` tolerate an existing `.graph-wiki.yaml` that contains `version: 2` (plus arbitrary other keys like `repo:`) but no `plugins` key. Today the line `entry = next((p for p in data["plugins"] ...))` raises `KeyError: 'plugins'`, which broke `/graph-wiki:bootstrap` after a user (or upstream skill prompt) seeded a provisional manifest to satisfy the `resolve_wiki_and_repo()` chicken-and-egg in `detect_containers.py`.

Purpose: Heal sparse manifests in place so bootstrap is robust to the documented "seed a provisional `.graph-wiki.yaml` first" workaround. We intentionally do NOT touch `detect_containers.py` or change the canonical schema — `plugins` remains required by the v2 schema; this is purely defensive on the write path.

Output: 3-line code change in `init.py` + one regression test in `test_init.py`.
</objective>

    """Returns v2 dict; raises on version < 2. Normalises `plugin` block but
