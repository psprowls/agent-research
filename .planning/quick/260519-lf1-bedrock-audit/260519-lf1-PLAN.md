# Quick Task 260519-lf1: Bedrock model audit script

## Description

Single Python utility at `scripts/bedrock_model_audit.py` that produces a JSON
report of every TEXT-output, ON_DEMAND Bedrock foundation model plus every
inference profile in a region, annotated with live tool-calling capability and
AWS Pricing API per-token pricing. Output drives sweep-candidate selection.

## Why

`aws bedrock list-foundation-models` doesn't expose tool-use capability and
pricing lives in a separate API entirely. Doing this once by hand, then keeping
the JSON in the repo, beats grepping AWS docs every time a new model lands.

## Scope (1 task)

### T1 — Write scripts/bedrock_model_audit.py

**Files:**
- `scripts/bedrock_model_audit.py` (new)

**Behaviour:**
1. `aws bedrock list-foundation-models` (filter: `byOutputModality=TEXT`, `byInferenceType=ON_DEMAND`)
2. `aws bedrock list-inference-profiles`
3. Concatenate both lists; for each entry:
   - **Probe:** `bedrock-runtime converse` with a minimal toolConfig (one tool, trivial schema) and a prompt that should trigger the tool. ~50 input / 20 output tokens. Detect success by either a `toolUse` content block or a clean response with stop reason. Failures classified:
     - `ValidationException` mentioning "tool" → `toolCalling=false`
     - `AccessDeniedException` → `toolCalling=null`, `probeError="AccessDenied"`
     - Other `ClientError` / `BotoCoreError` → `toolCalling=null`, `probeError=<short class name>`
     - Clean response with toolUse → `toolCalling=true`
   - **Pricing:** `aws pricing get-products --service-code AmazonBedrock --filters ...` keyed on the model's underlying foundation model name. Extract `Inp-1K-Tokens` and `Otp-1K-Tokens` SKUs. Pricing fetch runs regardless of probe outcome.
4. Concurrency: `asyncio.Semaphore(8)` over probe calls. Pricing fetches run in a separate pool of the same size.
5. Output: `[{...AWS catalog record..., toolCalling, probeError?, pricing}]` to `--out` (default `bedrock-models.json`).
6. Final summary line: `Wrote N models to <path>; X tool-calling, Y priced, Z access-denied`.
7. `--dry-run` flag: print model IDs that would be probed and exit 0 without invoking anything.

**CLI:**
- `--region` (default `us-east-1`)
- `--out` (default `bedrock-models.json`)
- `--concurrency` (default 8)
- `--dry-run`

**Dependencies:** boto3 (already a transitive workspace dep via langchain-aws).
Use boto3 directly with `asyncio.to_thread` for blocking calls — no aiobotocore
to avoid an extra dependency.

**Verify:**
- `uv run python scripts/bedrock_model_audit.py --dry-run` prints a non-empty model list and exits 0.
- (Optional, live, costs <$0.50:) `uv run python scripts/bedrock_model_audit.py` produces a valid JSON array file with `toolCalling` and `pricing` keys on each record.

**Done:** Script exists, executes successfully in `--dry-run` mode, contains all four CLI flags, and writes valid JSON.

## must_haves

- `scripts/bedrock_model_audit.py` exists and is executable via `uv run python`
- `--dry-run` mode does not make probe or pricing API calls
- Output JSON preserves the original AWS catalog fields and adds `toolCalling` + `pricing` keys
- Access-denied probes are non-fatal and produce `toolCalling: null, probeError: "AccessDenied"`
- Concurrency capped via `asyncio.Semaphore`
